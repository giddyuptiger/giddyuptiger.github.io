## Change 1 — Goals UI + in-app editing (commit c48f989)

### Display on project cards (`projectCard`)
- Each card now shows, between the short-status line and the state/urgency footer:
  - the project's **`smart_goal`** (flag icon, two-line clamp),
  - a **progress bar** when `goal_target` is set — `goal_current / goal_target goal_unit` + a
    clamped 0–100% fill in the primary green (`goalProgressBar`),
  - a **blocker chip**: amber `block` chip with the blocker text when a real blocker exists, or a
    green "No blockers" chip when the field is empty or starts with "None" (`blockerActive`).

### Editing (writes back to Supabase under the user's own RLS)
- The project detail modal (`projectModal`) gains a **"Goal & blocker" panel** (`goalModalSection`)
  showing the goal, progress bar and blocker, plus an **Edit** affordance.
- `goalEditModal` is a form with all five fields (SMART goal, blocker, current, target, unit).
  `saveGoalEdit` mutates the project object in place and persists via **`writeProjectsCol`** —
  `sb.from("maestro_dashboard").update({ projects: DATA.projects }).eq("user_id", USER_ID)` — the
  **identical, already-proven write path** the app uses for task edits (`writeTasksCol`). Numeric
  fields are coerced with `parseFloat`; blank target/current/unit delete the key. So Jeremy can
  edit goals and blockers straight from the project list, no Claude round-trip.

### Annual North Star (editable banner)
- New **North Star banner** at the top of the Overview (`northStarBanner`) — star icon, year, the
  guiding-goal text, and an edit pencil. `northStarModal` edits text + year; `saveNorthStar`
  writes `settings.northStar = {text, year}` via the existing settings write path.
- Reads tolerate either a `{text, year}` object or a bare string (`northStarData`).

### Wiring
- Four new delegated actions added to the existing project/task click handler: `edit-goal`,
  `goal-save`, `edit-northstar`, `ns-save`. Matches the app's existing modal/popover + "…"/eye
  interaction patterns.

## Change 2 — Remove the Routines tab (commit 3a8dfd9)
- Dropped the **Routines** link from the desktop nav and the mobile bottom-nav.
- Removed the `#view-routines` section and the `renderRoutines()` view function.
- Removed `"routines"` from both router view allow-lists and from the render-fn map — **no dead
  refs**. Nav rebalances automatically (desktop tabs 8→7, mobile bnav 7→6; mobile items are
  `flex-1`).
- **Backing data left untouched**: the event-kind mapping (`routine`/`meditation`) stays, so those
  events still surface in Overview/Week/Travel — only the Routines *tab/view* was dropped.
